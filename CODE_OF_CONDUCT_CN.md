# 🤝 行为准则 · Code of Conduct

[🇬🇧 English](CODE_OF_CONDUCT_EN.md)

---

## 我们的承诺

Gary CLI 是一个面向嵌入式开发者的开源项目。我们致力于为每一位参与者——无论其技术水平、背景或经验深浅——营造一个开放、友善、无骚扰的社区环境。

参与本项目的所有人，包括贡献者、维护者和用户，都应遵守本行为准则。

---

## 我们倡导的行为

- **互相尊重**：对不同的技术方案、工具偏好和使用习惯保持开放态度
- **建设性反馈**：在 Code Review、Issue 讨论中聚焦于代码和问题本身，而非针对个人
- **耐心与包容**：嵌入式开发门槛较高，欢迎各个阶段的学习者提问，不嘲笑基础问题
- **诚实透明**：如实描述 Bug、限制和已知问题，不夸大功能，不隐瞒已知缺陷
- **感谢贡献**：每一份 PR、Issue、Skill 包和文档改进都值得被认可

---

## 我们不接受的行为

- 针对个人的侮辱、嘲讽或人身攻击
- 任何形式的骚扰行为，无论公开讨论还是私信
- 未经许可发布他人的私人信息（真实姓名、邮箱、地址等）
- 与项目无关的政治、宗教或其他争议性内容
- 任何形式的歧视性言论

---

## 适用范围

本准则适用于所有项目相关空间，包括：

- GitHub Issues、Pull Requests、Discussions
- 项目相关的社交媒体讨论
- 任何以项目名义参与的线上或线下活动

---

## 执行与报告

如果你目睹或遭遇了违反本准则的行为，请联系维护者：

- **GitHub 私信**：[@PrettyMyGirlZyy4Embedded](https://github.com/PrettyMyGirlZyy4Embedded)
- **私密渠道**：对于涉及隐私或严重骚扰的情况，可通过 [Security Advisory](https://github.com/PrettyMyGirlZyy4Embedded/garycli/security/advisories/new) 私下反馈

所有投诉都将被认真对待并严格保密。维护者将根据情节轻重作出如下处理：

| 级别 | 情形 | 处理方式 |
|------|------|----------|
| 1 · 提醒 | 用词不当、无意冒犯 | 私下提醒，说明问题所在 |
| 2 · 警告 | 持续不当行为或单次较严重违规 | 正式警告，说明后果 |
| 3 · 临时封禁 | 严重违规或警告后仍不改正 | 限定期限内限制参与项目 |
| 4 · 永久封禁 | 持续骚扰或极端不当行为 | 永久移除参与资格 |
