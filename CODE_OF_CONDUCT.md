# 🤝 Code of Conduct

[🇨🇳 中文版](CODE_OF_CONDUCT.md)

---

## Our Pledge

Gary CLI is an open source project built for embedded developers. We are committed to making participation a welcoming, friendly, and harassment-free experience for everyone — regardless of technical background, skill level, or experience.

Everyone who participates in this project, including contributors, maintainers, and users, is expected to follow this Code of Conduct.

---

## Encouraged Behavior

- **Mutual respect**: Stay open to different technical approaches, tool preferences, and workflows
- **Constructive feedback**: In Code Reviews and Issue discussions, focus on the code and the problem — not the person
- **Patience and inclusivity**: Embedded development has a steep learning curve; beginners are welcome to ask questions without fear of ridicule
- **Honesty and transparency**: Accurately describe bugs, limitations, and known issues — don't oversell features or conceal known problems
- **Acknowledging contributions**: Every PR, Issue, Skill pack, and documentation fix deserves recognition

---

## Unacceptable Behavior

- Personal insults, mockery, or ad hominem attacks
- Harassment of any kind, whether in public discussions or private messages
- Publishing someone's private information (real name, email, address, etc.) without their consent
- Off-topic political, religious, or otherwise divisive content unrelated to the project
- Discriminatory language or conduct of any kind

---

## Scope

This Code of Conduct applies across all project spaces, including:

- GitHub Issues, Pull Requests, and Discussions
- Social media discussions related to the project
- Any online or in-person events held in the project's name

---

## Enforcement and Reporting

If you witness or experience behavior that violates this Code of Conduct, please contact the maintainers:

- **GitHub**: Reach out to [@PrettyMyGirlZyy4Embedded](https://github.com/PrettyMyGirlZyy4Embedded) directly
- **Private channel**: For privacy-sensitive or serious harassment situations, use the [Security Advisory](https://github.com/PrettyMyGirlZyy4Embedded/garycli/security/advisories/new) to report privately

All reports will be taken seriously and handled confidentially. Maintainers will respond with actions proportional to the severity of the incident:

| Level | Situation | Response |
|-------|-----------|----------|
| 1 · Reminder | Inappropriate wording, unintentional offense | Private note explaining the issue |
| 2 · Warning | Repeated misconduct or a single significant violation | Formal warning with stated consequences |
| 3 · Temporary ban | Serious violation or continued behavior after warning | Restricted participation for a set period |
| 4 · Permanent ban | Sustained harassment or extreme misconduct | Permanent removal from the project |
